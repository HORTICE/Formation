ADOTECH.BJ

Depuis 2021;le groupe ADOTECH spécialisé dans la digitalisation a pour but de propulser d'avantage l'humanité vers le numérique et ceux grace aux formations mises en place pour aider la jeune estudiantine à apprendre la programmation web afin de developper des competences à générer soit meme l'interface complète d'un site web à partir des languages du Front end que sont le HTML et le CSS et enfin le JAVASCRIPT ; un language de gestion de front end et back end .

Comme dit précédemment;le programme de formation mis en place vise les débutants dans le domaine de la programmation et a pour objectifs majeurs de faciliter le premier contacte des interessés avec le developpement web et donner les bases pour pouvour permetrre aux participants de générer leur propre interface. Le programme de la formation est schématisé suivant cette affiche.

  1. On appelle HTML(HyperText Markup Language 6) ; la dernière version de HTML concue pour représenter les pages web

      2. Nous avons besoin de ce part d'aborder les notions de:

    • Réalisation et Structure d'une page HTML:

      Le DOCTYPE qui permet de dire au navigateur comment lire le HTML et de construire l'arbre associé

      La balise head est l'entete de la page.Obligatoire pour un traitement aisé de la page par le navigateur

      Les balises META qui indiquent un certains nomnbre de paramètres permettant une prise en charge correcte de la page comme pare exemple le référencement par les moteurs de recherche;l'encodage utilisé.La balise title indique le titre de la page à l'affichage.A l'intérieur de la balise body ;nous mettons le contenu à afficher sur la page(texte;images;...en plus de leur mise en forme éventuelle)

      • Permet d'introduire de nouvelles possibilités notemment pour rendre le contenu web plus riche et interactif.

    • Avantage du HTML:

      Présence d'éléments sémantiques (les balises )

      La création des formulaires à base des balises input type..... notemment pour receuillir le nom ;l'email;les dates; numéro et pleins d'autres choses

      • Les balises structurent le contenu de la page(texte;images ;etc...).Chaque balise a un role et donne du sens au contenu présenté.

    • Syntaxe d'une page HTML: Balises;elements et attributs

      Certaines balises peuvent etre complètées par des attributs précisant certains paramètres(l'adresse des liens ;source d'une image à afficher;etc.)

      Une balise peut contenir plusieurs attributs.

    • Création et Organisatiion d'une page HTML

  2. En anglais CSS signifie Cascading Style sheets et se traduit par feuilles de styles en cascades

    • Role du CSS

      On utilise ce language pour décrire la présentation d'un document écrit en HTML ou XML .Le CSS décrit les couleurs et images de fond; le remplissage;les marges ;les bordures...Il permet égalementde rendre un document plus agréable à lire;mieux présenté.Il a pour role d'améliorer la mise en page;de séparer la structure de sa présentation et d'améliorer l'accessibilité des pages web

    • Insertion du code CSS

      Le code CSS est relié au code HTML via un lien dans le head du HTML

      C'est le cas le plus interessant. Si vos règles ne sont pas stockées dans une feuille de styleexterne;en cas de modifications de ces meme règles ;votre travail va vite devenir fastidiuex;surtout si elles sont présentes sur un grand nombre de pages HTML.

      Nous utilisons la balise style pour préciser le type et enfin la balise sortante pour fermer la balise style avec le caractère"/".

    • Règles du CSS

      Le CSS présente des règles formulées selon la synthaxe suivante: Sélécteur{propriétéCSS: attributs ou valeur}

      Quelques formes classiques de sélecteurs:
      1. Sélecteurs simples
      2. Sélecteurs multiples: Plusieurs balises séparées par des virgules.
      3. Sélecteurs contextuels : Plusieurs sélecteurs séparés par une espace.
      4. Sélecteurs Pseudo-Classe qui rajoute un contexte à un sélecteur.

      En dehors de ces sélecteurs nous aborderons d'autres sélecteurs dit sélecteurs personnalisés qui ont des particularités en HTML et CSS.
